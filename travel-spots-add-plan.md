## JSON 데이터 구조

각 여행 정보는 다음 구조를 따릅니다:

```json
{
    "name": "여행지 한글 이름",
    "english name": "English Name of the Place",
    "title": "짧은 매력적인 타이틀",
    "description": "상세한 설명 (2-3문장)",
    "city": "도시명",
    "province": "주(Province)명",
    "icon": "이모지 아이콘",
    "category": "카테고리 (해변/섬, 폭포, 다이빙 등)",
    "imageUrl": "대표 이미지 URL",
    "texts": [
        "# 마크다운 형식의 상세 정보 1",
        "## 방문 정보 마크다운 2",
        "## 추천 포인트 마크다운 3"
    ]
}
```

## 정보 검색 도구

Claude Code는 다음 도구들을 직접 사용하여 여행 정보를 검색하고 수집합니다:

### WebSearch 도구
- **용도**: 인터넷에서 여행지 정보를 검색
- **활용 예시**:
  - `"Cebu tourist attractions 2026"` - 세부 관광 명소 검색
  - `"Palawan best beaches Philippines"` - 팔라완 해변 검색
  - `"Bohol waterfalls travel guide"` - 보홀 폭포 가이드 검색
- **장점**: 최신 정보, 다양한 출처에서 종합적인 정보 수집

### WebFetch 도구
- **용도**: 특정 웹페이지에서 상세 정보 추출
- **활용 예시**:
  - 관광청 공식 사이트에서 운영 시간, 입장료 정보 수집
  - 여행 블로그에서 방문 팁, 추천 포인트 추출
  - Wikipedia에서 역사적 배경, 지리 정보 수집
- **장점**: 특정 페이지의 심층 정보 획득

### 검색 전략
```
1단계: WebSearch로 여행지 개요 파악
   └─> 여행지명 + "travel guide" 또는 "tourist information" 검색

2단계: WebFetch로 상세 정보 수집
   └─> 검색 결과 중 신뢰할 수 있는 페이지에서 상세 정보 추출

3단계: 정보 종합 및 검증
   └─> 여러 출처의 정보를 교차 검증하여 정확성 확보
```

### 단계별 상세 설명

#### STEP 1: 기존 여행지 목록 추출

```bash
# 전체 목록 출력
dart run lib/philgo_files/scripts/list_travel_spots_names.dart

# 파일로 저장 (AI에게 전달용)
dart run lib/philgo_files/scripts/list_travel_spots_names.dart > existing_spots.txt
```

#### STEP 2: Claude Code가 AI로서 직접 처리

> **핵심**: Claude Code 자체가 인공지능이므로, 별도의 AI에게 요청할 필요 없이 Claude Code가 직접 새로운 여행지를 찾아냅니다.

Claude Code가 다음 작업을 자동으로 수행:

1. **기존 목록 분석**: STEP 1에서 추출한 여행지 목록을 메모리에 로드
2. **AI 지식 활용**: Claude Code의 인공지능 지식을 활용하여 목록에 없는 필리핀 여행 명소 파악
3. **인터넷 검색**: 필요시 웹 검색을 통해 추가 정보 수집
4. **중복 필터링**: 기존 목록과 비교하여 중복 제거

```
예시 요청 (사용자 → Claude Code):
"세부(Cebu) 지역의 유명 여행 명소 중 기존 목록에 없는 곳 30개를 찾아주세요.
단, 산/트레킹 카테고리는 제외해주세요."

→ Claude Code가 AI로서 직접 분석하고 결과 제공
```

#### STEP 3-4: 검색 및 중복 확인

```bash
# 특정 여행지가 이미 존재하는지 확인
dart run lib/philgo_files/scripts/list_travel_spots_names.dart | grep -i "Kawasan"
dart run lib/philgo_files/scripts/list_travel_spots_names.dart | grep -i "카와산"
```

#### STEP 5: JSON 추가 시 필수 확인사항

1. 모든 필수 필드 존재 확인
2. imageUrl이 유효한 URL인지 확인
3. texts 배열에 마크다운 형식 정보 포함
4. JSON 파일 전체 유효성 검증

## 작업 시 주의사항

### 제외 대상 (반드시 확인)
1. ❌ **산/트레킹 카테고리** - 추가하지 않음
2. ❌ **민다나오 지역** - 다바오, 수리가오, 카가얀데오로 등 제외

### 중복 방지
1. 추가 전 반드시 `name` 또는 `english name` 중복 체크
2. 비슷한 이름의 다른 장소도 구분 필요 (예: "화이트 비치"는 여러 곳에 존재)
3. 도시/주 정보로 구분

### 데이터 품질
1. 모든 필수 필드 채우기
2. imageUrl은 유효한 URL 사용
3. texts 배열은 마크다운 형식으로 상세 정보 포함
4. 한글과 영문 이름 모두 정확하게 기입

### 안전한 작업
1. 작업 전 백업 생성
2. 배치 단위로 저장 (30개마다)
3. JSON 유효성 검증 후 저장
